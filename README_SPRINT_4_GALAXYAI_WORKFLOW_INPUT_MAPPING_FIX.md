# Sprint 4 GalaxyAI Workflow Input Mapping Fix

## Issue

Rudy provided the exact Python request generated by GalaxyAI:

```python
data = {
    "workflowId": "cmp4j5u8o0001jm049rtesgiw",
    "values": {
        "node_1772800705319_request": {
            "Car prompt": "your text here"
        }
    }
}
```

The VIP app was sending a generic `prompt` value, so GalaxyAI started the workflow but did not use the approved VIP prompt correctly.

## Fix

This patch updates:

```text
src/lib/galaxyai/client.ts
src/app/api/galaxyai/runs/route.ts
```

The VIP app now sends the approved GalaxyAI prompt using the exact field structure expected by this workflow:

```json
{
  "workflowId": "cmp4j5u8o0001jm049rtesgiw",
  "values": {
    "node_1772800705319_request": {
      "Car prompt": "APPROVED VIP PROMPT HERE"
    }
  }
}
```

## Apply

1. Replace `src/lib/galaxyai/client.ts`.
2. Replace `src/app/api/galaxyai/runs/route.ts`.
3. Commit.
4. Push.
5. Let Vercel redeploy.

Suggested commit message:

```text
Map VIP prompt to GalaxyAI workflow input
```

## Test

1. Open a campaign.
2. Approve a `galaxyai_prompt` asset with a very obvious prompt.
