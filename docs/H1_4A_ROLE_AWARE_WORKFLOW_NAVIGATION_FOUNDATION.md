# H1.4A Role-Aware Workflow Navigation Foundation

## Purpose

Clean up VIP navigation around real workflow roles:

- MASTER/platform users should be able to see all accounts and all platform-level tools.
- Account owners/admins should only see account-scoped workflows they can use for their active account.
- Navigation labels should match user workflow language instead of internal implementation names.

## Files changed

```text
src/lib/accounts/account-context.ts
src/app/(app)/layout.tsx
src/components/layout/AppShell.tsx
src/components/layout/SidebarNav.tsx
```

## What changed

- Adds platform role and active account role to the shared account context.
- Treats `master`, `owner`, `admin`, `platform_owner`, and `platform_admin` as MASTER-style platform roles for now.
- MASTER users can load all non-archived accounts into the account switcher.
- Account users only receive their owned/member accounts.
- Sidebar navigation is now generated by role/context.
- MASTER-only pages are hidden from account users.
- Account users get an account-scoped workflow nav.
- Renames confusing publishing nav labels:
  - `Publishing Schedule` becomes `Publish Center`.
  - `Actions` becomes `Action History`.
  - `Zapier` becomes `Integrations` and is MASTER-only.
  - `Publishing Ready` is removed from primary navigation because it is an asset-specific execution page, not a queue.
  - `Ready Queue` is removed from primary navigation for now to avoid confusion.

## Important security note

This patch improves navigation visibility and user experience. It does not replace server-side authorization or RLS. Hidden links are not security by themselves. H1.4C should add/verify route and API guards so account users cannot access other accounts by typing URLs manually.
